# 📝 Классификатор текстовых отзывов

Автоматический скрипт на Python для определения тональности текстовых отзывов. Поддерживает два режима работы: машинное обучение (TF-IDF + Logistic Regression) на размеченных данных и лексический анализ по словарям ключевых слов.

## ✨ Возможности

| Функция | Описание |
|---------|----------|
| 🔍 **Два режима классификации** | ML-модель (scikit-learn) или keyword-based подход |
| 📊 **Автоопределение режима** | Скрипт сам выбирает ML, если найдена разметка `true_label` |
| 🌐 **Мультиязычность** | Поддержка русского и английского языков |

📂 Формат входных данных
Скрипт принимает CSV-файл с обязательной колонкой review_text. Колонка true_label опциональна и нужна только для обучения ML-модели.
Пример 1: Без разметки (keyword-based режим)
csv
Copy

review_text
"Отличный товар, быстрая доставка!"
"Ужасное качество, не рекомендую"
"Скажите, а есть ли доставка в Калининград?"
"Всё работает, спасибо"
"Товар сломался на второй день"

Пример 2: С разметкой (ML-режим)
csv
Copy

review_text,true_label
"Отличный товар, быстрая доставка!","позитивный"
"Ужасное качество, не рекомендую","негативный"
"Скажите, а есть ли доставка в Калининград?","нейтральный"
"Всё работает, спасибо","позитивный"
"Товар сломался на второй день","негативный"

Требования к разметке для ML

    Минимум 2 класса
    Минимум 5 примеров на каждый класс (по умолчанию)
    Рекомендуется ≥20 примеров для оценки на тестовой выборке

⚙️ Режимы работы
Режим 1: ML (машинное обучение)
Активируется автоматически, если в CSV найдена колонка true_label с достаточным количеством примеров.
Алгоритм:

    Векторизация: TF-IDF (униграммы + биграммы)
    Модель: Logistic Regression с балансировкой классов
    Предобработка: нижний регистр, очистка пробелов

Оценка качества:

    Accuracy (общая точность)
    Classification Report (precision, recall, f1-score)
    Confusion Matrix (матрица ошибок)

Режим 2: Keyword-based (лексический)
Активируется, если разметки нет или её недостаточно.
Принцип работы:

    Подсчёт позитивных, негативных и нейтральных слов в тексте
    Учёт многословных фраз (вес ×2)
    Поддержка русского и английского языков

Классы:
Table
Класс	Признаки
позитивный	Преобладание позитивных слов
негативный	Преобладание негативных слов
нейтральный	Вопросительные конструкции или равенство счётчиков
📤 Выходные данные
CSV-файл
Создаётся рядом с исходным (суффикс _classified):
Table
Колонка	Описание

true_label	Истинная метка (если была)
predicted_label	Предсказанная тональность
Кодировка: UTF-8-BOM (корректно открывается в Excel).
Консольный вывод

    Количество и процент по каждому классу
    Accuracy и матрица ошибок (если есть true_label)
    Путь к сохранённому файлу

🔧 Настройка
Основные параметры
В начале скрипта classify_reviews.py:
Python
Copy

INPUT_CSV_PATH = "reviews.csv"           # Путь к входному файлу
OUTPUT_CSV_PATH = None                   # Путь к выходному (None = авто)
TEXT_COLUMN = "review_text"              # Название колонки с текстом
LABEL_COLUMN = "true_label"              # Название колонки с метками
PREDICTED_COLUMN = "predicted_label"     # Название новой колонки
MIN_SAMPLES_PER_CLASS = 5                # Мин. примеров для ML
TEST_SIZE = 0.2                          # Доля тестовой выборки
RANDOM_STATE = 42                        # Зерно для воспроизводимости

🛠️ Устранение неполадок
Table
Проблема	Решение
Файл не найден	Проверьте INPUT_CSV_PATH и текущую директорию (os.getcwd())
Отсутствует колонка 'review_text'	Переименуйте колонку в CSV или измените TEXT_COLUMN
UnicodeDecodeError	Скрипт автоматически пробует cp1251, но лучше сохраните файл в UTF-8
Недостаточно данных для ML	Добавьте примеры в CSV или уменьшите MIN_SAMPLES_PER_CLASS
Excel показывает кракозябры	Выходной файл сохраняется в UTF-8-BOM — Excel должен открывать корректно
Пустые строки в выходе	Скрипт автоматически удаляет их; проверьте исходный CSV
